CDN, Dominios y Cache

Cómo funciona la entrega de assets en SIRV y cómo manejar el cache.

El CDN de SIRV maneja tiempos de cache largos por defecto. Aunque reemplaces un archivo, el browser o CDN puede seguir sirviendo la versión anterior durante horas o días. No asumir que un reemplazo se verá de inmediato.

Cómo forzar actualización inmediata

Cambiar el nombre del archivo

La única forma segura de que un cambio se vea inmediatamente es subir el archivo con un nombre diferente. Esto genera una nueva URL, un nuevo cache y carga inmediata del recurso.

Correcto
banner-hero-January.webp
→ Nueva URL → carga inmediata
No garantiza cambio
banner-hero-enero.webp
→ Misma URL → puede quedar cacheado
La carpeta /ecommerce/banners tiene cache más corto y mayor flexibilidad para cambios. Está diseñada para campañas de alta rotación. Alkosto usa el dominio especial ads.dam.alkosto.com para banners comerciales.
